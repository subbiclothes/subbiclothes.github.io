Chat Commands

Power-user shortcuts and text commands — useful for gestures or fast execution without opening menus. Every command has a menu equivalent; nothing here is required to use the HUD.

Channels

Which channel you can use depends on your privilege level:

ChannelWhoNotes
/55Owner onlyStandard channel — basic and advanced commands. Also works from an object owned by the owner, message prefixed with sc:.
/65VIP / MastersThe owner, an actively-granted admin, anyone on the Masterlist, or anyone whose access category has Adv.Commands enabled. Broadly equivalent in power to /55.
/45External / remoteAnyone, only if "Remote Access" is enabled and their category has the Remote permission. Message must be prefixed with the owner's legacy name, e.g. Jane Doe wear.outfit MyLook.
/75Creator onlyAdministration channel, reserved for the creator and hardcoded admin UUIDs. Maintenance/support tooling — not relevant to end users, not documented here.

Permission Command Syntax

perm.<category>.<name>[=<value>]

Categoriespub/public, wl/whitelist, fr/friends, gr/group
Namesclothes, outfits, accessories, hairstyle, nudity, remote, random, distance, huds, adv.commands/advcmd, hid.folders/hidfld

Public toggles without needing a value; the rest need =on/=off explicitly. remote controls access to the /45 external channel; random controls access to Random Outfit/Avatar.

Notes on Specific Commands

clothes.lock / .hardlock / .unlock — currently a WIP feature, roleplay/submission-themed, not a technical lock against third parties. clothes.lock [part]: anyone who isn't the owner/master sees the option greyed out as [LOCKED] and it does nothing if touched — the owner can still interact with that part normally. clothes.hardlock [part]: same lock effect, but also applied to the owner themself — though the owner can disable it any time via a direct button, so it isn't a real restriction, just a roleplay affordance.

scfind [-v] — WIP, in active development for the DEV-4.1 line. Broadcasts a message to the whole region/parcel; other Subbi Clothes HUDs with the option enabled respond with <username> - {gender} - {bio} - {custom message} (gender, bio, and message are optional; bio only shows if set). Intended for finding people to interact/roleplay with in the region.
