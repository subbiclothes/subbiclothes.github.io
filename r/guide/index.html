Overview

What is Subbi Clothes?

Subbi Clothes is a complete RLV Wardrobe HUD for Second Life. It lets you manage outfits, swap avatars, control who can interact with your wardrobe, and trigger animations, sounds, and particle effects on every clothing change — all from a single wearable.

It works with any mesh body, any avatar type, and any clothing system including BOM layers. It is not tied to any specific body brand.

Immersion

Animations, sounds, and particles on every outfit change.

Interactivity

Let others dress, undress, or swap your avatar with full permission control.

Security

PG-Safe+ automatically protects you in restricted regions or while AFK.

Customization

Colors, messages, animations, gender profiles — everything configurable per avatar.

Before you start

Requirements

Subbi Clothes is an RLV HUD. It requires a viewer with RLVa support and RLV enabled. It will not function in viewers that do not support RLV.

RequirementDetails
Viewer Firestorm 7.2 or higher is strongly recommended. Any viewer with full RLVa support will work.
RLV enabled In Firestorm: Avatar → Avatar Health → Allow Remote Scripted Viewer Controls (RLVa)
Inventory access The HUD reads your #RLV folder. Make sure your viewer has this folder and RLV is on.
Permissions The HUD is Copy · Mod · Transfer. No extra purchases required.

Setup

Quick Start

From unboxing to wearing your first outfit in 4 steps.

1

Get the HUD

Purchase Subbi Clothes on the Second Life Marketplace. Accept the folder delivery and locate it in your inventory.

2

Create your folder structure with SAF v3

Rez the SAF v3 object included in the package. It will automatically build the entire #RLV/SubbiClothes/ folder tree in your inventory. What used to take hours is done in minutes.

3

Add your avatars and outfits

Move your avatar folders and outfit items into the #RLV/SubbiClothes/ structure created by SAF. Firestorm's inventory is recommended for managing RLV folders.

4

Configure with the Outfit Config Editor

Use the web-based Config Editor to set up animations, colors, particle effects, and permissions for each outfit. Export and paste the result into your config notecard.

Feature

Clothing Management

Subbi Clothes manages your clothing at the layer level. Each body part (top, bottom, shoes, etc.) can have multiple states: dressed, underwear, undressed, or any custom state you configure.

How it works

Clicking a body part cycles through its states. You can also trigger states by name via chat commands or through interactions from other avatars (if permitted).

Quick Dress / Quick Undress

These shortcuts apply a state change to all body parts at once — useful for events, changing rooms, or RP scenes.

Clothing states are defined per outfit in the config notecard. Use the Config Editor to set them up visually.

Feature

Avatar & Outfit Swap

Group your saved avatars and outfits inside #RLV/SubbiClothes/. Subbi Clothes reads these folders and lets you switch between them instantly from the HUD menu.

Features

  • Instant swap — switch to a full avatar (body + clothes) in one click.
  • Random pick — let the HUD pick a random avatar or outfit from a group.
  • Mix & match — combine body from one avatar with clothes from another.
  • Shared access — let specific people swap your avatar (with permission).

Folder structure

SAF v3 creates the correct structure automatically. Avatars go in their own subfolder. The HUD auto-detects folders on scan.

#RLV/ SubbiClothes/ Avatars/ My Catgirl/ Human v2/ Outfits/ Casual/ Formal/

Feature

Accessories System

The accessories system lets you attach and detach individual items separately from your main outfit — glasses, bags, props, collar, tail, etc.

Key behaviors

  • Items are auto-detected from your accessories subfolder.
  • Supports subfolders for grouping related items.
  • Auto-detach on outfit change — accessories can be set to detach when you switch avatars or outfits.
  • Accessories can be toggled individually from the HUD menu.

Feature

Access Control & Permissions

You decide who can interact with your HUD, and what they can do. Permissions are set per category — clothes, avatars, accessories, hairstyle, nudity, and more.

Permission levels

LevelWho it applies to
PublicAnyone in the same sim.
GroupAnyone in your active group.
FriendsAnyone on your friends list.
WhitelistSpecific avatars you've manually approved.
BlacklistSpecific avatars always blocked regardless of other settings.

Interaction distance

Set a minimum distance for interactions. Avatars outside that range cannot trigger actions even if they have the right permission level.

The Whitelist and Blacklist override all other permission levels. Blacklisted avatars can never interact regardless of group or friend status.

Feature

PG-Safe+ Mode

PG-Safe+ is an automatic protection system that triggers when you enter restricted regions or go AFK. When active, it can:

  • Dress your avatar to a defined "safe" outfit.
  • Revoke permissions from anyone currently controlling your HUD.
  • Replace your look entirely with a fallback avatar.

Modes

ModeBehavior
AutoTriggers automatically based on region rating and AFK status.
ManualTriggered only when you activate it from the HUD menu.
OffDisabled entirely for this outfit.

PG-Safe+ mode is configured per outfit via the Config Editor.

Tool

Outfit Config Editor

The Outfit Config Editor is a free web tool (no install required) for configuring your outfits visually. It replaces the need to edit config notecards by hand.

What you can configure

Body Parts & States

Define which clothing states each body part supports (dressed, underwear, undressed, custom).

Animations

Set wear/remove animations, timing, and looping per outfit.

Colors & Particles

Color pickers with gradient presets, particle system settings.

Permissions

Default permission levels per outfit — can be overridden by global settings.

Import / Export

Load an existing config notecard into the editor with the Load button. When done, click Export to copy the finished config — paste it into your notecard in-world.

Tool · Included with HUD

SAF v3 — Semi-Automatic Folders Creator

SAF v3 is an in-world object included with the HUD package. Rez it on the ground and it automatically creates the entire Subbi Clothes folder structure inside your #RLV inventory.

Running SAF v3 is Step 2 of the Quick Start. Do it once after purchasing. You don't need to run it again unless you want to reset your folder structure.

After SAF runs, your inventory will have a ready-to-use folder tree. You only need to move your avatar and outfit items into the correct subfolders.

FAQ

Common Questions

Does it work with my body / avatar?

Yes. Subbi Clothes works with any mesh body, any avatar type, and any clothing — including BOM layers and alpha fixes. It is not locked to any specific body brand or system.

Is RLV required?

Yes. Subbi Clothes is an RLV HUD and requires a viewer with RLVa support. Firestorm 7.2 or higher is strongly recommended. You also need to have RLV enabled in your viewer settings.

Do I need scripting knowledge?

No. SAF v3 creates all folders automatically. The Outfit Config Editor handles configuration visually. Initial setup takes 10–15 minutes following this guide.

Can others interact with my HUD?

Yes, with full control on your end. You choose per category (clothes, avatars, accessories, hairstyle, nudity…) what each person — public, friends, group, or individual whitelist — can and cannot do.

What version is current?

The latest public release is v1.3.3 B4D. A new major version (DEV) is in active development. Owners can request redelivery via /55 redelivery in-world.

How do I update?

Say /55 redelivery in-world near the original vendor or a redelivery terminal to receive the latest version at no extra cost.
