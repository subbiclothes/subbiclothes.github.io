Chat CommandsComandos de Chat

Power-user shortcuts and text commands — useful for gestures or fast execution without opening menus. Every command has a menu equivalent; nothing here is required to use the HUD. Atajos y comandos de texto para usuarios avanzados — útiles para gestos o ejecución rápida sin abrir menús. Todo comando tiene un equivalente en el menú; nada de esto es obligatorio para usar el HUD.

Permission Command SyntaxSintaxis de Comandos de Permisos

perm.<category>.<name>[=<value>]

CategoriesCategoríaspub/public, wl/whitelist, fr/friends, gr/group
NamesNombresclothes, outfits, accessories, hairstyle, nudity, remote, random, distance, huds, adv.commands/advcmd, hid.folders/hidfld

Public toggles without needing a value; the rest need =on/=off explicitly. remote controls access to the /45 external channel; random controls access to Random Outfit/Avatar. distance (Public category) is the same underlying setting as "Min.Range" in the Settings menu — changing either one updates both. Public se activa/desactiva sin necesitar un valor; el resto necesita =on/=off explícitamente. remote controla el acceso al canal externo /45; random controla el acceso a Outfit/Avatar Aleatorio. distance (categoría Public) es el mismo dato que "Min.Range" en el menú de Settings — cambiar uno actualiza el otro.

Notes on Specific CommandsNotas sobre Comandos Específicos

clothes.lock / .hardlock / .unlock — per-bodypart lock. clothes.lock [part]: anyone who isn't the owner/master sees the option greyed out as [LOCKED] and it does nothing if touched — the owner can still change that part normally. clothes.hardlock [part]: same, but also blocks the owner via a real RLV restriction until unlocked again — this is enforced technically, not just a greyed-out menu option. Locks are per-outfit: they reset automatically whenever you change avatar/outfit, so they're never accidentally "stuck" on a new look. clothes.lock / .hardlock / .unlock — bloqueo por parte del cuerpo. clothes.lock [parte]: cualquiera que no sea el dueño/master ve la opción en gris como [LOCKED] y no hace nada si se toca — el dueño puede seguir cambiando esa parte con normalidad. clothes.hardlock [parte]: igual, pero también bloquea al propio dueño mediante una restricción RLV real hasta que se desbloquea de nuevo — se aplica técnicamente, no es solo una opción en gris del menú. Los bloqueos son por-outfit: se reinician automáticamente al cambiar de avatar/outfit, así que nunca se quedan "pegados" por accidente a un look nuevo.

avatar.lock / .hardlock / .unlock — a separate, whole-avatar lock (independent from the per-bodypart one above). While active: "Avatars" disappears from both your own menu and the external menu, and every avatar/outfit-change command (@@, outfit.wear/.replace/.change/.back/.random/.menu) is blocked with a warning. It persists through script resets, and — by design — has no menu button of its own; it's command-only, so it can't be toggled by mistake from a menu. avatar.lock / .hardlock / .unlock — un bloqueo distinto, de avatar completo (independiente del bloqueo por parte del cuerpo de arriba). Mientras está activo: "Avatars" desaparece tanto de tu propio menú como del menú externo, y todos los comandos de cambio de avatar/outfit (@@, outfit.wear/.replace/.change/.back/.random/.menu) quedan bloqueados con un aviso. Persiste tras reinicios de script, y — a propósito — no tiene botón de menú propio; es solo por comando, para que no se pueda activar/desactivar sin querer desde un menú.

Dynamic titles — title.set accepts two special formats instead of plain text. #SEQ[:tN]#phrase1|phrase2|... rotates through phrases one at a time; #SCROLL[:tN][:lN]#text scrolls the text sideways like a ticker. Optional parameters go between the mode name and the closing #, separated by :, in any order: t<N> sets the speed (ticks between advances — roughly 10 ticks per second), l<N> sets the visible window width, #SCROLL only. Example: title.set #SCROLL:t3:l25#Welcome to my HUD!. Both formats still support the {outfit}, {avatar}, {me} and {range} placeholders, same as a static title. Títulos dinámicos — title.set acepta dos formatos especiales en vez de texto plano. #SEQ[:tN]#frase1|frase2|... rota entre frases una a una; #SCROLL[:tN][:lN]#texto desplaza el texto en horizontal como una cinta. Los parámetros opcionales van entre el nombre del modo y el # de cierre, separados por :, en cualquier orden: t<N> fija la velocidad (ticks entre avances — aproximadamente 10 ticks por segundo), l<N> fija el ancho de la ventana visible, solo para #SCROLL. Ejemplo: title.set #SCROLL:t3:l25#¡Bienvenido a mi HUD!. Ambos formatos siguen admitiendo los placeholders {outfit}, {avatar}, {me} y {range}, igual que un título estático.

invite <name> — the name must match someone physically present in the same region (legacy or display name); the HUD sends them a clickable link on the /45 external channel that opens your menu for them, same as if they'd touched your HUD/Clicker directly — still subject to your normal access permissions. invite <nombre> — el nombre debe coincidir con alguien físicamente presente en la misma región (legacy o display name); el HUD le manda un link pulsable por el canal externo /45 que les abre tu menú, igual que si hubieran tocado tu HUD/Clicker directamente — sigue sujeto a tus permisos de acceso normales.

sc.reset <script> — resets a single HUD script by name, without detaching/reattaching the whole HUD. Useful if one part gets stuck. Valid names: MAIN, Settings, HUD, Updater, Accessories, MenuProcessor, Menu, Outfits, Outfits_RLV, RLV, Memory, Perms, CmdConfig, Commands (the last one resets the command script itself). sc.reset <script> — reinicia un único script del HUD por nombre, sin tener que desadjuntar/readjuntar el HUD entero. Útil si una parte se queda colgada. Nombres válidos: MAIN, Settings, HUD, Updater, Accessories, MenuProcessor, Menu, Outfits, Outfits_RLV, RLV, Memory, Perms, CmdConfig, Commands (este último reinicia el propio script de comandos).

No value = toggle — for settings.<opt>, access.<opt> and perm.<category>.<name>, leaving off =<value> flips the current state instead of doing nothing — e.g. settings.otheranims with no value toggles it on/off. Same idea as the Public permission category, extended to the rest of the config commands. Sin valor = toggle — en settings.<opt>, access.<opt> y perm.<categoría>.<nombre>, omitir =<valor> invierte el estado actual en vez de no hacer nada — p. ej. settings.otheranims sin valor lo activa/desactiva. Misma idea que la categoría de permiso Public, extendida al resto de comandos de configuración.

<bool> values — anywhere a command takes a boolean (title.show, title=, particles., particles=, permission toggles, etc.), any of these work: on/off, true/false, 1/0, yes/no — case-insensitive. Valores <bool> — en cualquier comando que acepte un booleano (title.show, title=, particles., particles=, toggles de permisos, etc.), sirve cualquiera de estos: on/off, true/false, 1/0, yes/no — sin distinguir mayúsculas/minúsculas.
